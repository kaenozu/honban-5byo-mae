# 本番5秒前

テレビ生放送のディレクターとして、90秒間に発生する放送事故をカメラ切替・CM・ミュート・テロップで処理する Godot 4.4 向け縦切りプロトタイプです。

## 現在の内容

- 1280×720 固定、16:9
- 90秒のタイムライン
- 3台のカメラとオンエア画面
- データ駆動の事故イベント（`data/events.json`）
- 初期4事故: 声量巨大化、強風、馬乱入、鍋爆発
- スコア、視聴率、事故度、CM残数・クールダウン
- キーボード操作と画面ボタン

## 操作

| 操作 | キー |
| --- | --- |
| CAM1へ切替 | `1` |
| CAM2へ切替 | `2` |
| CAM3へ切替 | `3` |
| CMを使用 | `C` |
| ミュート切替 | `M` |
| テロップ切替 | `T` |

## 実行

1. Godot 4.4 でこのフォルダの `project.godot` をインポートします。
2. `F6` または `F5` で実行します。

CLI版GodotがPATHにある場合は次でも起動できます。

```bash
godot --path .
```

## 構成

- `scripts/GameManager.gd`: 唯一のゲーム状態管理元
- `scripts/EventManager.gd`: JSONタイムラインとイベント進行
- `scripts/ScoreManager.gd`: 成否の得点処理と最終結果
- `scripts/Main.gd`: UI入力、Signal接続、画面更新
- `data/events.json`: 事故イベント定義
- `data/ui_tokens.json`: レイアウト、色、文字、モーションの設計トークン

## 現段階の制約

カメラ映像は外部アセットを使わない色面プレースホルダーです。ゲームループと事故対応の手触りを先に検証するための縦切り版であり、映像演出・SE・本番用アニメーションは後続実装を想定しています。
